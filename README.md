# WebSecPlayground - セキュア認証実装版

このプロジェクトは、**JWT（JSON Web Token）ベースのセキュアな認証・認可機能**を備えた Next.js ウェブアプリケーションです。
元の `web-sec-playground-2` リポジトリをベースとしつつ、不要なセッションベース認証や脆弱なコードを完全に排除し、堅牢なセキュリティ対策を施しました。

## セキュアな設計と創意工夫

本アプリは、以下のセキュアな設計原則に基づいて実装されています。

1. **パスワードの安全なハッシュ化保存 (bcrypt)**
   パスワードは平文で保存せず、`bcryptjs` を用いてハッシュ化（ソルト付き）してからデータベースに保存しています。認証時も `bcrypt.compare` を用いて安全に検証しています。
2. **CSPおよびセキュリティヘッダの厳格化**
   `src/proxy.ts` において、`X-Frame-Options` や `X-XSS-Protection`、`Strict-Transport-Security`、そして厳格な `Content-Security-Policy` ヘッダを設定し、XSS などのクライアントサイド攻撃を軽減しています。

## 独自に実装した認証・認可に関する追加機能

認証・認可に関するセキュリティ強化機能として、以下の5つの機能を実装しました。

### 1. パスワード強度インジケーター
**実装箇所**: `src/app/signup/page.tsx`
ユーザーが新規登録（サインアップ）する際、入力したパスワードの強度（長さ、大文字小文字、数字、記号の組み合わせ）を評価し、リアルタイムでメーター表示する機能を実装しました。これにより、ユーザーに推測されにくい強力なパスワードの設定を促します。

![パスワード強度インジケーター1](./docs/images/01-password-strength1.png)
![パスワード強度インジケーター2](./docs/images/01-password-strength2.png)
![パスワード強度インジケーター3](./docs/images/01-password-strength3.png)
*(画像: 強度ゲージが動的に変化し、ユーザーに安全なパスワード設定を促すUI)*

### 2. パスワードの表示・非表示切り替え機能
**実装箇所**: `src/app/login/page.tsx`, `src/app/signup/page.tsx`
パスワード入力欄の右端に目のアイコンを配置し、クリックすることで入力したパスワードの文字を確認できるようにしました。これにより、複雑なパスワードを設定・入力する際のタイプミスを防ぎ、ユーザビリティとセキュリティの両立を図ります。

![パスワード表示・非表示機能1](./docs/images/02-password-toggle.png1.png)
![パスワード表示・非表示機能2](./docs/images/02-password-toggle.png2.png)
*(画像: 目玉アイコンをクリックして伏せ字を解除した状態)*


### 3. パスワードの変更機能
**実装箇所**: `src/app/member/password/page.tsx` (フロントエンド) / `src/app/api/password/route.ts` (バックエンドAPI)
ログイン済みのユーザーが、任意のタイミングで自身のパスワードを変更できる機能です。バックエンド API では、現在使用中のパスワードを `bcrypt` で検証した上で、新しいパスワードを再度ハッシュ化して安全に更新します。

![パスワード変更画面1](./docs/images/03-change-password1.png)
![パスワード変更画面2](./docs/images/03-change-password2.png)
*(画像: 現在のパスワード確認と、新しいパスワードを二重入力させる変更画面)*


### 4. ログイン試行の間隔制限（レートリミット）機能
**実装箇所**: `src/app/api/login/route.ts`
ブルートフォース攻撃やパスワードスプレー攻撃を防ぐため、同一のメールアドレスに対して連続して5回パスワードを間違えると、そのアドレスに対するログインを15分間ブロックするレートリミット機構をバックエンドAPIに実装しました。

![レートリミットエラー表示](./docs/images/04-rate-limit.png)
*(画像: 上限回数に達した際に「しばらく経ってから再度お試しください」と警告するエラー画面)*


### 5. アカウント削除（退会）機能
**実装箇所**: `src/app/member/delete-account/page.tsx` (フロントエンド) / `src/app/api/user/route.ts` (バックエンドAPI)
情報保護（忘れられる権利）の観点から、ユーザー自身が自分のアカウントと関連データを完全に削除できる退会機能を実装しました。重要な破壊的変更であるため、UI上で警告を出し、実行時に再度現在のパスワードを要求する認可の仕組みを設けています。

![アカウント削除画面1](./docs/images/05-delete-account1.png)
![アカウント削除画面2](./docs/images/05-delete-account2.png)
*(画像: 警告メッセージとともに、本人の最終確認としてパスワード入力を求める退会画面)*
